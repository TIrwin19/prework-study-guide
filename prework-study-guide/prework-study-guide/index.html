HTML

  • "head" element contains information about the webpage.
  • "body" element represents the visible content shown to the user.
  • "h1, h2, h3, h4, h5, and h6" tags represent header text size rank.
  • "p" tag represents text in a paragraph format.
  • "li" tag represents a list text format.
  • "ul and ol" tags represent unordered and ordered list format.
  • "br" tag creates a line of empty space or break between two blocks of content.
  • "img" tag contains information about images displayed on a webpage.
    1. "src=" contains the link to the image being displayed.
    2. "alt=" contains an short descriplion of the image.
    3. "width=" contains the with desired for the image being displayed.
    4. "height=" contains the height desired for the image being displayed.
  • "a" tag stands for anchor and creates a hyperlink.
    1. "href" contains a URL for the link and resides within the opening "< >".
    2. The text to be used for the hyperlink falls between the opening and closing tags.
  • "footer" tag appears at the bottom of a webpage and contains author, copyright, contact, sitemap, and navigation info.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • "*": apply rules to all elements on page
  • External style sheets must be linked to the HTML file.
  • "style": tag used in HTML to apply a style
  • A "." before a class designates it as a class selector.

Git

  • "git status": checks what branch we are currently on
  • "git checkout main": puts you in the main branch
  • "git pull origin main": pulls down the latest main branch
  • "git checkout -b 'branch-name'": creates a new branch and switches to it
  • "git add -A": stage all files in working branch to be commited
  • "git commit -m 'descriptive text'": commit staged files to repo
  • :git push origin 'branch name'": push all changes up to Github
  • "code .": opens current file in VScode
